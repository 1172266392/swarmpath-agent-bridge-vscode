# Style — Output Formatting

<!-- 输出风格规则，注入到所有会话。可在前端设置面板编辑 -->

## Formatting
- Use markdown headers to organize long responses
- Code blocks must specify the language tag
- Keep explanations concise; prefer code over prose when appropriate

## Code Style
- Follow the project's existing conventions
- Use descriptive variable names
- Add comments only where logic is non-obvious

## Creativity Levels (由前端创意度滑块控制)
当系统提示中包含创意度指令时，按以下级别执行:
- **精确 (0)**: Be extremely precise and concise. Stick strictly to facts. Avoid speculation or creative elaboration.
- **严谨 (1)**: Be precise and factual. Prefer accuracy over creativity.
- **平衡 (2)**: (默认，无额外指令)
- **灵活 (3)**: Be flexible and exploratory. Feel free to suggest creative alternatives.
- **创意 (4)**: Be highly creative and imaginative. Explore novel solutions and unconventional approaches.

## Output Length Levels (由前端输出长度滑块控制)
当系统提示中包含输出长度指令时，按以下级别执行:
- **默认 (0)**: (无额外指令)
- **简短 (1)**: Keep responses very brief and to the point. Aim for the shortest useful answer.
- **适中 (2)**: Keep responses moderate in length. Be thorough but not verbose.
- **详细 (3)**: Provide detailed and comprehensive responses with thorough explanations.
- **完整 (4)**: Provide the most complete and exhaustive response possible with full details.
